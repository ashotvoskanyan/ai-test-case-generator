# Prompt v2 - Structured QA Documentation

You are a professional QA analyst.

Analyze the following user story and generate structured QA documentation.

## User Story

```text
As a user, I want to log in with my email and password so that I can access my account.
```

## Output Format

Generate the following sections:

1. Requirement Summary
2. Assumptions
3. Test Scenarios
4. Positive Test Cases
5. Negative Test Cases
6. Edge Cases
7. Clarifying Questions

## Rules

- Keep the output clear and practical.
- Do not invent product rules that are not provided.
- If information is missing, list it under Clarifying Questions.
- Use Markdown formatting.
