# Login API Test Cases

## API Endpoint Description

```text
POST /login

Request body:
{
  "email": "user@example.com",
  "password": "Password123"
}

Expected behavior:
If the email and password are valid, the API returns a successful response with an authentication token.
```

## Prompt Used

API Test Prompt v1

## AI Output

## 1. API Summary

The `/login` endpoint allows a user to authenticate by sending an email and password. If the credentials are valid, the API should return a successful response with an authentication token.

## 2. Assumptions

- The endpoint uses the `POST` method.
- The request body requires `email` and `password`.
- The API validates both email and password.
- A successful login returns an authentication token.
- Invalid credentials should not return a token.
- Passwords and sensitive data should not be exposed in the response.

## 3. Test Scenarios

- User logs in successfully with valid credentials.
- User tries to log in with an incorrect password.
- User tries to log in with an unregistered email.
- User submits an empty request body.
- User submits a request with missing email.
- User submits a request with missing password.
- User submits an invalid email format.
- API does not expose sensitive data in the response.

- API should handle repeated failed login attempts according to security rules.
- API should use HTTPS in real environments.
